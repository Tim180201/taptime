# Domain Model

Status: Initial Draft

## Known Domain Concepts

| Concept | Description | Status |
|---|---|---|
| User | Person using the system. | Draft |
| Role | Permission category assigned to a user. | Draft |
| NFC Tag | Physical chip used to identify or trigger a time tracking action. | Draft |
| Scan Event | Raw event generated by scanning an NFC tag. | Draft |
| Time Entry | Business record representing tracked time. | Draft |
| Assignment | Relationship between NFC tag and domain entity or action. | Draft |
| Organization | Business/customer context. | Open |
| Project/Job | Trackable work context. | Open |
| Export | Generated reporting artifact. | Open |

## Domain Rule

Raw NFC scan events and business time entries must not be treated as the same concept.

A scan is evidence of an interaction. A time entry is a validated business record.

## Open Modeling Questions

- Does one NFC tag represent a user, a workplace, a project, or an action?
- Can one scan start and stop a timer?
- Are time entries created instantly or after validation?
- What happens when a scan is duplicated, invalid or offline?
